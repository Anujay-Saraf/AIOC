## Infrastructure

### Database Clusters
- pg-primary: Main RDS Postgres 14 — us-east-1a, Multi-AZ enabled
- pg-replica-01/02: Read replicas for reporting and non-transactional reads
- Max connections: 500 (shared across all services)
- Connection pool best practice: pool_size = (DB max_connections / num_services / num_pods) - 5

### Cache Clusters
- redis-cluster (payments): 6-node cluster, 64GB, eviction=allkeys-lru
- redis-cluster-search: 3-node cluster, 32GB, eviction=volatile-lru

### Kubernetes
- Cluster: prod-eks-us-east-1 (EKS 1.28)
- HPA enabled on all services: min=2, max=20 pods
- PodDisruptionBudget: minAvailable=1 for critical services

## Common Root Causes and Their Signatures

| Root Cause | Key Log Pattern | Primary Metric |
|-----------|----------------|---------------|
| DB pool exhaustion | "Connection pool exhausted" | db_pool_utilization > 90% |
| Cache stampede | "cache miss rate 95%" | cache_hit_rate < 10% |
| Retry storm | "retry amplification" | upstream_rps / service_rps > 3x |
| Memory leak | "GC pause > 2000ms" | jvm_heap_used > 85% |
| Deployment regression | Error spike after deploy | error_rate delta vs pre-deploy |
| Downstream degradation | "circuit breaker OPEN" | downstream_latency_p99 > timeout |
